## The problem RAG solves

LLMs have a knowledge cutoff and no access to your private data. RAG bridges that gap by retrieving relevant documents at query time and feeding them as context.

## The basic pipeline

1. **Chunk** your documents into meaningful pieces
2. **Embed** each chunk into a vector
3. **Store** vectors in a vector database (Pinecone, Chroma, pgvector)
4. At query time: **embed the question**, find similar chunks, **inject into the prompt**

```python
from langchain_community.vectorstores import Chroma
from langchain_openai import OpenAIEmbeddings

vectorstore = Chroma.from_documents(docs, OpenAIEmbeddings())
retriever = vectorstore.as_retriever(search_kwargs={"k": 4})
```

## When to use RAG vs fine-tuning

Use RAG when your data changes frequently or you need source attribution. Fine-tune when you need the model to adopt a specific style or behaviour consistently.

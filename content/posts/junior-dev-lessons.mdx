## Ship ugly, then iterate

My first instinct was to perfect everything before sharing. That was wrong. Getting feedback early — even on rough work — is worth more than polished work nobody has seen.

## Read the source code

Documentation tells you what. Source code tells you how and why. When something confusing, I started reading the actual library code. It changed how I debug and learn.

## Understand before you copy

Stack Overflow and AI tools make it easy to paste solutions you don't understand. That debt compounds. If I can't explain every line, I rewrite it until I can.

## Build things you actually use

The projects I learned the most from were tools I built because I was annoyed at something. Real problems make better teachers than tutorial projects.
